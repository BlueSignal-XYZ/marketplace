# Weekly Vault Maintenance

Run this routine once per week (suggest: Monday morning before Start of Day).

---

## Tasks

### 1. Archive Completed Drafts
- Move finalized drafts from `Projects/drafts/` to `Projects/archive/`
- Rename with date prefix: `YYYY-MM-DD-Original-Title.md`
- Update any Dashboard references

### 2. Clean Inbox
- Verify all `Inbox/` items have been processed
- Delete or archive any stale items
- If items have been sitting for 7+ days, flag them for user attention

### 3. Review Stale TODOs
- Scan `Dashboard.md` for tasks older than 14 days
- Flag stale items for the user: should they be re-prioritized, delegated, or removed?
- Update timestamps on items that are still active

### 4. Update Knowledge Files
- Review `Knowledge/Training/Feedback-Log.md` for recurring themes
- Distill new patterns into `Knowledge/Training/Learned-Patterns.md`
- Check `Knowledge/BlueSignal/` files — update if project details have changed

### 5. People Directory
- Add any new contacts encountered during the week
- Update existing contact files with new context

### 6. Vault Health Check
- Verify all wiki-links in CLAUDE.md point to existing files
- Check for orphaned files (files not referenced anywhere)
- Ensure naming conventions are followed (Hyphenated-Title-Case.md)

---

## Output

After maintenance, provide a brief summary:
- Items archived
- Stale TODOs flagged
- Knowledge files updated
- Any issues found during health check
