# First-Run Setup

Run this routine when `Dashboard.md` does not exist or the vault has just been created.

---

## Checklist

### 1. Personalize CLAUDE.md
- [ ] Update the "Who I Am" section with your name and role
- [ ] Review Core Principles — adjust if any don't fit your workflow
- [ ] Update Preferences section with your communication style

### 2. Configure Projects
- [ ] Review [[Knowledge/BlueSignal/WQM-1]] — correct any inaccuracies
- [ ] Review [[Knowledge/BlueSignal/Marketplace]] — correct any inaccuracies
- [ ] Add any additional projects to `Knowledge/BlueSignal/`

### 3. Add People
- [ ] Add key team members to `Knowledge/People/Team/`
- [ ] Add partners/vendors to `Knowledge/People/Partners/`
- [ ] Use format: one file per person, `Firstname-Lastname.md`

### 4. Configure Connected Tools
- [ ] Set up GitHub MCP integration
- [ ] Set up email integration (if available)
- [ ] Set up calendar integration (if available)
- [ ] Update the "Connected Tools" section in CLAUDE.md

### 5. Set Initial Training Preferences
- [ ] Add any known preferences to `Knowledge/Training/Custom-Instructions.md`
- [ ] Example: coding style preferences, communication preferences, project conventions

### 6. Create Dashboard.md
- [ ] Generate the initial Dashboard.md with priority-sorted tasks
- [ ] Include any pending items from the setup process

### 7. Create Today.md
- [ ] Generate the first Today.md with today's focus and tasks

---

## After Setup

Once complete, the assistant should:
1. Confirm all files were created successfully
2. Summarize what was configured
3. Ask if anything needs adjustment
4. Begin normal operations using the Start of Day routine
